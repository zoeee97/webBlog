                # vue+node项目打包优化并部署上线+nginx配置

                # 一、配置config.js进行vue打包优化配置

                用到的优化手段有：cdn加速、公共代码抽离、代码压缩、图片文件压缩。 因此需要先下载的依赖有：

                //代码压缩
npm install uglifyjs-webpack-plugin --save-dev
//图片压缩
npm install image-webpack-loader --save-dev

                总的config.js配置文件如下：

                const UglifyJsPlugin = require('uglifyjs-webpack-plugin')
// 是否为生产环境
const isProduction = process.env.NODE_ENV !== 'development';

// 本地环境是否需要使用cdn
const devNeedCdn = false

// cdn链接
const cdn = {
    // cdn：资源名:外部引入名（外部引入名由模块自身决定，不可更改）
    externals: {
        'vue': 'Vue',
        'vue-router': 'VueRouter',
        'axios': 'axios',
        'lodash': '_',
    },
    // cdn的css链接
    css: [

    ],
    // cdn的js链接
    js: [
        'https://cdn.bootcss.com/vue/2.6.11/vue.min.js',
        'https://cdn.bootcss.com/vue-router/3.2.0/vue-router.min.js',
        'https://cdn.bootcss.com/axios/1.3.4/axios.min.js',
        'https://cdn.bootcdn.net/ajax/libs/lodash.js/4.17.21/lodash.min.js'
    ]
}

module.exports = {
  assetsDir: 'static',     //  outputDir的静态资源目录
  publicPath: './',   // 静态资源路径（默认/
  productionSourceMap: false, //不输出map文件
  chainWebpack: config => {
    // 发布模式
    config.plugin('html').tap(args => {
        // 生产环境或本地需要cdn时，才注入cdn
        if (isProduction || devNeedCdn) args[0].cdn = cdn
        return args
    })

    // 图片压缩
    config.plugins.delete('prefetch')
    config.module.rule('images')
        .test(/\.(png|jpe?g|gif|svg)(\?.*)?$/)
        .use('image-webpack-loader')
        .loader('image-webpack-loader')
        .options({ bypassOnDebug: true })

},

configureWebpack: config => {
    if (isProduction || devNeedCdn) config.externals = cdn.externals
    // 代码压缩
    config.plugins.push(
        new UglifyJsPlugin({
            uglifyOptions: {
                //生产环境自动删除console
                compress: {
                    drop_debugger: true,
                    drop_console: true,
                    pure_funcs: ['console.log']
                }
            },
            sourceMap: false,
            parallel: true
        })
    )

    // 公共代码抽离
    config.optimization = {
        splitChunks: {
            cacheGroups: {
                vendor: {
                    chunks: 'all',
                    test: /node_modules/,
                    name: 'vendor',
                    minChunks: 1,
                    maxInitialRequests: 5,
                    minSize: 0,
                    priority: 100
                },
                common: {
                    chunks: 'all',
                    test: /[\\/]src[\\/]js[\\/]/,
                    name: 'common',
                    minChunks: 2,
                    maxInitialRequests: 5,
                    minSize: 0,
                    priority: 60
                },
                styles: {
                    name: 'styles',
                    test: /\.(sa|sc|c)ss$/,
                    chunks: 'all',
                    enforce: true
                },
                runtimeChunk: {
                    name: 'manifest'
                }
            }
        }
    }
},
  devServer: {
    // 前端页面端口号配置
    port: 4000,
    // 自动打开浏览器
    open: true,
    // 注意：打包后webpack在本地配置的代理服务器会失效！！
    proxy:{
      '/api': {
        target: '你的后端接口地址',
        changeOrigin: true,
        secure: false, 
        pathRewrite: {'^/api': ''}
      }
    }
  }
}

                # 代码抽离

                将代码分离到不同的bundle中，之后可以按需加载或并行加载这些文件。 默认情况所有js代码在首页都会全部加载，就会影响首屏加载速度；代码抽离就可以分出更小粒度的bundle，控制资源加载的优先级。 因为webpack默认已经安装和集成了splitChunksPlugin，所以直接配置即可，详情看之前的config.js中的配置。

                # 最后

                最后npm run build打包得到dist文件夹，再在本地浏览器中打开index.html，如果页面正常显示则说明成功了。

                # 二、项目上传到服务器

                后端node项目直接上传整个文件夹到服务器，前端项目则只上传dist文件夹。 之后服务器和宝塔面板安全组都要开放对应的前端页面端口和后端端口。

                在宝塔面板-网站处添加Node项目。

                # 三、nginx配置代理

                server
    {
        # 用于监听的端口,配置为原vue项目启动端口
        listen 8021;
        server_name localhost;
        # 项目首页
        index index.html index.htm index.php;
        root /www/server/tomcat/webapps;
        #root  /www/server/phpmyadmin;
            location ~ /tmp/ {
                return 403;
            }

        #error_page   404   /404.html;
        include enable-php.conf;
        
        #监听路由名
        location /chart-demo
        {
          #vue包所在目录
          alias /www/server/tomcat/webapps/chart-demo/dist;
        }
        
        # 代理名称
        location /api
        {
          # 目标URL
          proxy_pass  http://127.0.0.1:8082/api;
          # 发送域名
          #proxy_set_header Host www.xxx.com;
        	# 内容替换
        	#sub_filter "/api" "";
        }

        access_log  /www/wwwlogs/access.log;
    }
# nginx配置文件所在路径
include /www/server/panel/vhost/nginx/*.conf;
}

                用Nginx配置代理如上文，像我这里的配置，我直接访问http://服务器ip:8021/chart-demo/open in new window 就可以访问到我们前端项目。proxy_pass配置和vue项目中的proxy里的target地址一样即可。
